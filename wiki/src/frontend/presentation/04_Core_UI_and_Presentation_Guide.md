# 04. Core UI and Presentation Guide

### 1. 핵심 UI 구현 내용
*   **미니멀 럭셔리 브랜딩**: 다크 모드 기반의 배경 위에 절제된 타이포그래피와 글래스모피즘(Glassmorphism) 효과를 적용하여 고급스러운 분위기를 연출하였습니다.
*   **동적 사용자 경험**: AOS(Animate On Scroll) 라이브러리를 적용하여 스크롤 위치에 따라 요소들이 부드럽게 나타나는 애니메이션을 구현하였습니다.
*   **비주얼 일관성 확보**: AI로 생성한 모든 제품 이미지 에셋은 **Eye-Level Front View, 85mm 렌즈 시점, Top-Left 조명, 일관된 배경색**이라는 엄격한 촬영 규격(Specs)을 통일하여 브랜드의 비각적인 일관성을 극대화하였습니다.

### 2. 발표 시 강조 포인트
*   **통합 아키텍처 설계**: 단순한 화면 구현을 넘어, 프론트엔드의 `config.js` 설정부터 Django 백엔드 데이터 통신, NGINX 리버스 프록시를 통한 서비스 배포까지 이어지는 전체 인프라 통신 아키텍처를 완결성 있게 설계하였습니다.
*   **논리적 제약 해결의 시각화**: 사용자의 복잡한 취향 충돌(예: 특정 패션 스타일 선호와 향기 불호의 상관관계) 및 알레르기 제약 조건을 논리적으로 해결하고, 이를 레이더 차트 및 분석 리포트 UI로 명확하게 시각화하여 정보 전달력을 높였습니다.

### 3. 주요 Q&A 방어 가이드
*   **Q. React Native 대신 웹(React)을 선택한 이유는?**
    - **A**: Capacitor 프레임워크를 활용하는 전략을 채택했습니다. 이를 통해 단일 웹 코드베이스로 iOS와 Android 앱을 동시에 런칭할 수 있는 크로스 플랫폼 환경을 구축하여 개발 생산성을 높이고 유지보수 효율을 극대화했습니다.
*   **Q. 데이터 수집 출처와 신뢰성은?**
    - **A**: 단순 웹 크롤링이 아닌 Kaggle의 전문 오픈 데이터셋(FragDB)과 실제 브랜드 성분 데이터를 혼합하여 데이터베이스를 구축했습니다. 이를 RAG(검색 증강 생성) 파이프라인에 결합함으로써 AI의 할루시네이션(환각 현상)을 방지하고 검증된 향수 정보만을 제공합니다.
