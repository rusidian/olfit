# 03. CSS and Tailwind CSS

### 1. CSS 개념
CSS(Cascading Style Sheets)는 HTML로 작성된 뼈대에 디자인을 입히는 언어입니다. 폰트, 색상, 레이아웃, 간격 등 웹 페이지의 시각적인 모든 요소를 정의합니다.

### 2. Tailwind CSS 도입 이유
*   **파일 관리 효율성**: 별도의 CSS 파일을 생성하여 클래스 이름을 고민하고 관리하는 번거로움이 없습니다.
*   **충돌 방지**: 각 요소에 고유한 유틸리티 클래스를 적용하므로, 클래스명 중복으로 인해 스타일이 꼬이는 문제를 근본적으로 해결합니다.
*   **개발 속도 극대화**: HTML 태그 내에서 `flex`, `p-4`, `text-center`와 같이 직관적인 클래스명을 사용하여 스타일을 즉시 적용할 수 있습니다.

### 3. 반응형 설계 (Responsive Design)
*   **Mobile-First 전략**: 모바일 기기에 최적화된 화면을 먼저 설계한 뒤, 화면 크기가 커짐에 따라 레이아웃을 조절하는 방식을 사용합니다.
*   **유연한 대응**: Tailwind CSS의 반응형 접두사(`sm:`, `md:`, `lg:`)를 활용하여 웹과 모바일 앱 환경을 동시에 지원하는 일관된 사용자 인터페이스를 신속하게 구현하였습니다.
